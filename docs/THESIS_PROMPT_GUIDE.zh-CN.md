# 毕设提示词工程指南

## 1. 项目定位

本项目是一个面向毕业设计场景的智能垃圾识别与回收建议系统，整体采用前后端分离与推理服务解耦的结构：

1. 前端：`Vue 3 + Vite + Tailwind CSS`
2. 后端：`Spring Boot + JPA + PostgreSQL`
3. 推理服务：`FastAPI + Ultralytics YOLO`
4. 业务流程：图片上传 -> 垃圾检测 -> 检测结果展示 -> AI 回收建议 -> 多轮追问 -> 历史图库 -> 错误样本回流 -> 实验记录

在毕业设计场景中，提示词不应只服务于“聊天”，而应服务于以下三类任务：

1. 系统完善与功能优化
2. 毕业论文撰写与实验分析
3. 答辩材料准备与问题演练

因此，后续所有提示词都应遵循“贴合项目实际、符合论文规范、兼顾开发效率”的原则。

## 2. 提示词设计原则

### 2.1 论文导向原则

所有输出优先围绕本科毕业论文常见结构展开，包括摘要、绪论、需求分析、系统设计、系统实现、实验分析、总结与展望等内容，避免生成泛泛的产品介绍。

### 2.2 工程贴合原则

提示词应要求模型严格基于现有项目技术栈和模块结构进行分析，不脱离真实代码架构，不虚构不存在的功能。

### 2.3 可复用原则

提示词模板应尽量具备可替换性，例如只替换章节名、模块名或实验变量即可重复使用，以提高后续写作与开发效率。

### 2.4 结果可落地原则

对于功能优化建议，提示词应要求模型输出“优先级、实现难度、对论文的帮助”，便于在有限时间内选择高性价比任务。

## 3. 核心提示词模板

### 3.1 项目分析总控 Prompt

适用于让模型始终围绕当前毕设项目进行分析，避免输出空泛建议。

```text
你现在作为我的毕业设计指导助手，请基于一个“智能垃圾识别与回收建议系统”项目进行分析。

项目技术栈如下：
1. 前端：Vue 3 + Vite + Tailwind CSS
2. 后端：Spring Boot + Spring Data JPA + PostgreSQL
3. 推理服务：FastAPI + Ultralytics YOLO
4. 功能流程：图片上传 -> 垃圾目标检测 -> 检测结果展示 -> AI回收建议 -> 多轮追问 -> 图库管理 -> 错误样本回流 -> 实验记录

请严格遵守以下要求：
1. 所有分析必须贴合“本科毕业设计/毕业论文”场景
2. 输出风格偏正式、学术化，但不要过度空话
3. 结合软件工程和计算机论文常见结构来表达
4. 优先从“研究意义、系统功能、技术实现、实验验证、可改进点”五个维度分析
5. 如果提出建议，必须区分“可快速实现”和“适合写论文但实现成本较高”两类

请输出：
1. 该项目适合写进论文的核心创新点
2. 当前系统最值得强化的模块
3. 最容易被老师问到的薄弱点
4. 可直接用于后续写作与开发的建议
```

### 3.2 论文目录生成 Prompt

```text
请以本科计算机类毕业论文的规范格式，为题目“基于深度学习的校园垃圾识别与智能回收建议系统设计与实现”生成完整论文目录。

要求：
1. 目录符合本科毕业论文常见结构
2. 至少包含：摘要、Abstract、绪论、需求分析、系统总体设计、系统详细设计、核心功能实现、实验与结果分析、总结与展望、参考文献、致谢
3. 每一章下要给出二级标题，必要时给三级标题
4. 标题风格正式、规范、避免口语化
5. 目录必须结合我的项目实际功能：图片上传、YOLO识别、回收建议、多轮问答、历史图库、错误样本回流、实验记录

输出格式：
第一章 XXX
1.1 XXX
1.2 XXX
……
```

### 3.3 章节写作 Prompt

```text
请帮我撰写毕业论文中的“系统总体设计”章节，主题围绕“基于深度学习的垃圾识别与智能回收建议系统”。

写作要求：
1. 采用本科毕业论文的正式书面语言
2. 内容必须贴合真实系统，不要编造脱离项目结构的模块
3. 结合以下实际架构：
   - 前端：Vue 3
   - 后端：Spring Boot
   - 数据库：PostgreSQL
   - 推理服务：FastAPI + YOLO
4. 章节内容应体现软件工程思维，包含：
   - 系统总体架构设计
   - 功能模块划分
   - 数据流转过程
   - 前后端交互关系
   - 模型推理服务在系统中的位置
5. 用论文风格展开，不要用问答形式
6. 尽量避免“首先、其次、最后”机械重复，语言自然正式
7. 字数控制在 1200~1800 字

如果有必要，可附带：
1. 一张适合画成系统架构图的文字说明
2. 本章可以补充的图表建议
```

### 3.4 代码转论文描述 Prompt

```text
我现在有一个毕业设计项目，请你根据我提供的代码模块，帮我把工程实现内容转化为适合毕业论文的描述文字。

要求：
1. 不要照抄代码，不要写成开发文档
2. 要写成“论文中的系统实现描述”
3. 每一段都要体现模块职责、实现思路、交互逻辑
4. 语言正式、准确、简洁
5. 不夸大创新，不编造不存在的算法

我提供的模块是：
- 文件上传模块
- 垃圾检测模块
- AI建议生成模块
- 多轮对话模块
- 历史图库模块
- 错误样本标记模块

请按以下格式输出：
1. 模块功能说明
2. 实现思路
3. 关键流程
4. 可写入论文的表述版本
```

### 3.5 实验设计 Prompt

```text
请你作为我的毕业论文实验设计助手，围绕“基于YOLO的垃圾目标识别系统”设计实验方案。

我的项目背景：
1. 使用 YOLO 模型进行垃圾识别
2. 系统支持调整 imgsz、conf、iou 参数
3. 前端记录实验结果，后端保存识别信息
4. 目标是比较不同参数配置对识别效果和推理时延的影响

请你输出：
1. 实验目的
2. 实验环境配置
3. 数据集来源与处理说明
4. 实验变量设计（自变量、因变量、控制变量）
5. 对比实验表格模板
6. 实验结果分析写作模板
7. 如何从“准确率、召回率、mAP、推理时延、误检漏检”角度写分析

要求：
1. 符合本科毕业论文写法
2. 逻辑清晰
3. 不要写得像论文八股文，要像真实做过实验的人
```

### 3.6 数据集整理 Prompt

```text
请帮我从毕业论文角度撰写“数据集构建与预处理”内容。

我的数据来源主要包括：
1. 网络搜集的垃圾图片
2. 部分真实拍摄图片
3. 后续整理为适用于YOLO训练/测试的数据

请围绕以下问题展开：
1. 如何把“网络搜集图片”写得更规范、更学术
2. 如何描述图片筛选、去重、清洗、标注、类别归并
3. 如何说明训练集、验证集、测试集划分
4. 如何解释类别不平衡问题及处理方式
5. 如何用论文语言说明数据集存在的局限性

要求：
1. 输出内容适合直接写入毕业论文
2. 避免让人感觉数据来源不严谨
3. 实事求是，不要虚构特别大的数据规模
4. 语言正式，符合计算机专业论文风格
```

### 3.7 答辩问答 Prompt

```text
请你模拟毕业答辩老师，围绕我的课题“基于深度学习的垃圾识别与智能回收建议系统设计与实现”，给我提出 20 个高频问题，并分别给出参考回答。

要求：
1. 问题要贴合本科毕业答辩场景
2. 覆盖研究意义、技术选型、系统架构、模型选择、数据集来源、实验设计、系统不足、创新点等方面
3. 回答要控制在 100~200 字，适合现场口头表达
4. 回答风格自然、谦逊、逻辑清晰，不要过度夸大
5. 对于系统不足，要给出合理改进方向
```

### 3.8 开发优化 Prompt

```text
你现在作为我的毕业设计开发助手，请基于“垃圾识别 + 回收建议系统”的现有功能，帮我制定一个优先级明确的迭代计划。

要求：
1. 目标是兼顾“系统效果提升”和“论文可写性提升”
2. 必须区分：
   - 1周内可完成的优化
   - 适合论文展示的中等工作量优化
   - 时间不足时可以放弃的增强项
3. 每项优化都要说明：
   - 优化内容
   - 为什么值得做
   - 对论文哪一章有帮助
   - 实现难度
   - 性价比

请重点围绕以下模块提出建议：
1. 识别工作台
2. AI建议生成
3. 历史图库
4. 错误样本回流
5. 实验记录
6. 系统可视化展示
```

## 4. 高性价比使用场景

结合当前项目情况，提示词最适合优先用于以下任务：

1. 生成论文目录和章节提纲
2. 将现有代码逻辑转写为论文中的系统实现描述
3. 设计实验方案并整理实验结果分析模板
4. 规范描述网络图片采集、清洗和标注流程
5. 输出答辩高频问题与参考回答
6. 规划功能优化优先级，提升系统流畅度与可展示性

## 5. 万能母 Prompt

后续如需让模型长期围绕该项目稳定输出，可优先使用以下总控模板：

```text
你现在是“计算机毕业设计论文辅导助手 + 软件工程顾问 + 提示词工程专家”。

我的项目是一个“基于深度学习的垃圾识别与智能回收建议系统”，当前采用：

你的输出必须满足以下原则：

我接下来要你完成的任务是：
【在这里填你的具体任务，例如：帮我写系统总体设计章节】
```

## 6. 推荐使用顺序

为了提高整体效率，建议后续按以下顺序使用这些提示词：

1. 先生成论文目录与每章提纲
2. 再逐章生成正文初稿
3. 然后根据真实代码补充系统实现细节
4. 最后完成实验分析、摘要润色与答辩准备

按上述方式推进，可以让提示词工程同时服务于开发、论文与答辩，减少重复整理成本。
